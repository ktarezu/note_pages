学习如何对黄油进行逆向 ​

某一天，群里传了一个黄油的聊天记录，大概是一个我方女角色能被俘虏调教并且还有归还剧情的手游，并且在归还剧情中还要叫主角不要嫌弃她，即使她已经把敌方指挥官成为主人。我作为一个纯爱战士对此情此景感到很生气，想看一眼cg好好批判批判。但是把游戏下载之后发现居然是很眼熟的米池，并且怎么也找不到看归还剧情的入口，于是就有了这篇文章。

注意：本文章只作学习使用，禁止作为非法用途使用。

0.确定技术方案 ​

遇到黄油最简单的想法就是解包拿到资源。然后找到了这篇醒目的文章,通过文章提供的代码确实拿到了资源。可是将拿到资源发现它是spine。spine是啥玩意,黄油玩太少了只知道live2d。于是各种查找使用教程，发现使用错误版本的spine player就会直接报错，而拿了个最接近的版本，画面动起来了但人物没有脸，就只能换个思路了。

对于没逆向过的我来说，我对逆向的了解就是把编译后的可执行文件或者动态库拖到ida里面分析汇编。只学过单片机汇编的我看到X86汇编代码就头大，这么搞想要看到cg基本不可能。

然后通过那篇解包文章找到了一个关键字Il2CppDumper，再结合与天下布魔是同一个开发商，推测用的都是unity，最后生成il2cpp。那么就开始从il2cpp方面开始补习。

1.技术知识补习 ​

既然游戏用的unity引擎，那就要先对unity逆向有一定的了解。于是在吾爱破解上看到了这篇教程通杀爆改 Unity FPS 游戏系列。借用教程里的图:

首先Demo跟着过一遍，用cheat engine（下面简称CE）对游戏进行分析修改。不得不感叹CE确实强大，我刚开始对它的印象只有从内存中搜索值，并没有深入了解它的使用。它能够解析il2cpp的namespace、class、method，还能打断点，在不知道游戏在打开cg的时候调用哪个函数时一个一个打断点试试看就明白了。

当然还需要补习汇编、栈等方面的知识，举个例子:在X86机器中

c
int Add(int a, int b);
/**
参数：通常按从右到左的顺序依次压入栈中，所以在汇编代码中 b 会先压栈，然后是 a
返回值：存储在寄存器 EAX 中

调用 add(3, 5) 的汇编过程可能如下
push 5       ; 栈压入参数 b
push 3       ; 栈压入参数 a
call add     ; 调用函数
; 返回值在寄存器 EAX 中

 */

而在x86-64架构中

c
int Add(int a, int b);
/**
前几个参数（通常前 6 个整数参数）按顺序传递在寄存器中，a 会放在 EDI 寄存器中，b 会放在 ESI 寄存器中。
调用 add(3, 5) 的汇编过程可能如下
mov edi, 3    ; 将参数 a 传入 EDI
mov esi, 5    ; 将参数 b 传入 ESI
call add      ; 调用函数
; 返回值在 RAX 中

 */

现在大多数游戏都是64位的了，如果要修改调用函数的参数，就在Add函数被调用前用CE把寄存器edi改了就能实现改参数值的效果。

2.游戏分析 ​

下面以dmm pc版游戏为演示。

根据前面找到的关键字Il2CppDumper，去到这个项目，拿到最新的工具。这里我选择了Il2CppDumper-net6-win-v6.7.46。然后解压，保留这几个关键的文件。 从游戏目录获取GameAssembly.dll global-metadata.dat，为了方便起见，放到和工具同一个目录，然后建立一个目录output。如果想要复用工具可以设置环境变量，我觉得麻烦就dump一次烤一次工具， 然后在这个文件夹空白处按下shift+鼠标右键，在此处打开powshell，执行指令

powershell
.\Il2CppDumper.exe .\GameAssembly.dll .\global-metadata.dat .\output\

执行成功后，在output目录下就是我们想要的内容。

然后把GameAssembly.dll拖进ida，保持默认点确定就行。打开以后这个时候还不能急，可能要等十几分钟，左下角的状态变成idle以后，加载脚本选择第一个文件ida_py3.py，第二个选择output文件夹下的script.json，别选错成了config.json。

选择ida_py3.py 选择output文件夹下的script.json 当脚本执行完成以后，符号就还原完成了，这下就可以大概看懂游戏执行的大概流程了。

使用IDA也能调试，但是我更喜欢用CE，使用CE一个函数一个函数打断点以后，发现打开CG会调用函数UI_Memories_Cg.EnterCg(),在ida通过ctrl+p，然后ctrl+f找到EnterCg这个函数，双击跳转过去，就能看到进入CG的大概流程。

在游戏的具体实现中，在打开cg的时候服务端会验证cg是否已经开启，把GameAssembly.dll拖进ida分析后，发现函数名为GetMemoryAvgCanLook

c
//根据名称推测出这是一个验证成就cg能否查看的函数
void GetMemoryAvgCanLook(int avgId,void(*funcCallback)());

在上面的知识补习中，第一个参数是avgId，也就对应着在执行到这一条函数时edi的值就是avgId的值，可以打断点验证

在CE中选择游戏进程，然后在Momo-.Net Info中可以找到这个函数名：

双击函数就能看到函数所在的地址，右键对它打个断点，出现两次提示，都点yes

在游戏中打开海伦的回想，会触发断点，右边会有寄存器的信息，但是显示的是十六进制，把rdi用计算器或者python转换成10进制，可以看到这个寄存器正是31010103即海伦回想的avgId。意味着之后如果传入了未解放avgId的时候，只要把avgId强制改为31010103就能绕过服务端的验证。

不过和上面的理论有点差别的是，在ida中可以看到

asm
mov     rdx, rsi        ; funcCallback
mov     ecx, edi        ; avgId
call    MessageSender$$GetMemoryAvgCanLook

所以函数实际用到的寄存器实际是ecx，也就是第一个参数avgId实际上是ecx，通过上图可以看到ecx和edi的值一样，但需要更改ecx的才能使参数更改生效。简单验证方法就是双击ecx，改为一个未解放的avgId的十六进制，就会触发400错误

3.修改游戏逻辑 ​

3.1 GetMemoryAvgCanLook 绕过avgId认证 ​

上面得知改ecx就能成功更改avgId，在ida查看ecx在什么时候被使用

然后采用一个比较暴力的方式，直接偷梁换柱，在mov esi,ecx之前把ecx的值改为01D92D37(即31010103)，在ce中，通过代码注入的方式更改ecx。

选择地址，然后选择Auto Assemble 在弹出的窗口创建框架代码

选择注入代码 根据刚刚选择的地址，选OK 在这注入想要修改的代码

注入

asm
mov ecx,01D92D37

加入到CE 其它窗口会覆盖CE主窗口，把其它窗口缩小或关掉，可以看到在CE主界面会增加一项，勾选就能启用修改，双击改名为GetMemoryAvgCanLook好识别。

3.2 绕过GetChaInsultAvgCanLook ​

c
void GetChaInsultAvgCanLook(short chaId,int roomNum,void funcCallback)

与上面类似，分析过程就不赘述了，直接说结论，ecx是chaId,edx是roomNum 打断点，查看调教剧情，我这里初始是出云，默认调教开的是狐狸，可以看到默认角色id是1104(0x450),房间id自然也是0 直接上代码

asm
mov ecx,450
mov edx,0

3.3 绕过SendCheckHumiliateStateEnable ​

c
void SendCheckHumiliateStateEnable(int charId,uint state,int kind,...);

这个是在调教选择动作的时候会触发,charId->ecx,state->edx,kind->r8d

打断点，ecx=0x450(1104),edx=0x80(128),r8d=0x1 代码

asm
mov ecx,450
mov edx,80
mov r8d,1

3.4 开启全调教:GetPlayerFemdomRoomIdx ​

c
int GetPlayerFemdomRoomIdx(short charID)

这个函数比较特殊，我们只关心它的返回值，使用汇编指令retn，根据上面的补习理论，返回值会保存在rax或者eax中。于是事情就变得十分简单了。

在GData.GetPlayerFemdomRoomIdx处注入返回0x0a，不要问为什么，返回这个值就能解锁全部调教CG。

提示：GData.GetPlayerFemdomRoomIdx对CE来说就是一个地址。

asm
mov rax,0a
retn

效果如下，可以看到左边的回想还没解锁，这是正常的情况，因为调教和回想的解锁调教不一样。

3.5 开启全姿势:GetPlayerFemdomProgress ​

c
int GetPlayerFemdomProgress(short charID)

上面虽然解锁了全部的调教，但是姿势也要解锁，和上面一样，只关心函数返回值。

所以可以直接在GData.GetPlayerFemdomProgress处注入

注入代码如下

asm
mov rax,08
retn

结果是在GData.GetPlayerFemdomProgress处注入代码就可以开启全姿势。

3.6 开启全部好感剧情 GetPlayerCharAvgOpen ​

c
bool GetPlayerCharAvgOpen(...)

这里提示一下，rax,eax,ax,ah,al其实是表示同一个寄存器，只是范围不一样。bool类型只占1个字节，所以用al寄存器就够了。

3.8 dmm版去码及更改语言 ​

去码就在GData.get_CgMosaicFlag注入代码

c
bool get_CgMosaicFlag()

注入代码

asm
mov al,0 //0->false 1->true
retn

获取当前语言的函数为

c
int get_PlayLanguage()

改为简体中文在 GPlayerPrefs.get_PlayLanguage 注入代码

asm
mov rax,01 //0繁体中文 1简体中文 2英语 3日语 4韩语
retn

更改语言并不直接生效，需要到设置窗口触发更改语言。设置窗口有个静音按钮，借它来调用更改语言的函数。 调用函数会有点复杂，根据ida看到的函数，这两个都是同一个类下的函数，可以使用同一个this。

c
void __stdcall UI_Option__ChangeUILanguage(UI_Option_o *this, int32_t value, const MethodInfo *method);
void __stdcall UI_Option__ChangeMuteToggle(UI_Option_o *this, bool value, const MethodInfo *method);

思路就是UI在调用UI_Option__ChangeMuteToggle的时候调用UI_Option__ChangeUILanguage，通过注入实现下面的伪代码形式。

void __stdcall UI_Option__ChangeMuteToggle(UI_Option_o *this, bool value, const MethodInfo *method){
    UI_Option__ChangeUILanguage(this,1,method);//调用更改语言函数，1为简体中文
    ...
}

在UI_Option.ChangeMuteToggle注入代码。

asm
//备份寄存器
push rdi
push rsi
push rcx    ;this
//更改参数
mov rdi,1

//申请两个空间用来调用ChangeUILanguage
push rdi
push rcx
call UI_Option.ChangeUILanguage
//调用函数后清理刚刚申请的两个空间
add rsp,10

//恢复寄存器
pop rcx
pop rsi
pop rdi

然后在设置中点两次静音模式 弹出窗口，点右边确定 游戏会回到标题，进去游戏即是简体中文。

至此，游戏开CG结束。
